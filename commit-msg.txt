fix: AEGIS-TAURI-05 post-install bugs (double tray, IPC, window bounce)

Double tray icon:
  tauri.conf.json: removed trayIcon block — it auto-created a second tray icon
  alongside the one built in tray.rs. One source of truth: tray.rs only.

No data / IPC not wiring:
  tauri.conf.json: added withGlobalTauri:true — required for window.__TAURI__
  to be injected into the WebView. Without it, window.__TAURI__.event.listen
  doesn't exist and metrics events land nowhere.
  ui/index.html: updated waitForTauri() to check window.__TAURI__.event.listen
  directly (not __TAURI_INTERNALS__ which doesn't expose listen). Updated
  listen/invoke bindings to window.__TAURI__.event.listen and
  window.__TAURI__.core.invoke — the correct Tauri 2 global API surface.

Window bouncing on first open:
  main.rs: on_window_event CloseRequested now guards window.label() == "cockpit"
  to avoid firing on WebView2 internal init events from other contexts.
  main.rs: profile_changed emit delayed 1s after startup to give WebView time
  to register listeners before first event fires.
