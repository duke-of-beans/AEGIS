feat: AEGIS-TAURI-01 — Tauri 2 scaffold + Rust metrics core

Migrates AEGIS from Node.js/pm2/PowerShell-worker architecture to
Tauri 2 native desktop application. Zero elevation required for
CPU/RAM/disk/network/process monitoring.

src-tauri/ — full Tauri 2 Rust project:
  main.rs        — app entry, tray init, async runtime setup
  metrics.rs     — sysinfo polling (CPU/RAM/disk/network/processes)
                   emits "metrics" events to WebView every 2s
                   no subprocess, no PowerShell, no elevation
  commands.rs    — Tauri IPC commands: switch_profile, set_process_priority,
                   kill_process_cmd, get_intelligence, get_profiles
  profiles.rs    — YAML profile loader + apply via Win32 SetPriorityClass
  tray.rs        — native system tray with profile checkmarks
  sidecar.rs     — spawn + supervise intelligence sidecar binary

sidecar/         — intelligence layer scaffold (Sprint 03 populates)
  src/main.ts    — JSON-RPC entry point stub
  package.json   — pkg build config

ui/index.html    — minimal cockpit placeholder (Sprint 04 wires full UI)

tauri.conf.json  — window, tray, NSIS bundler, sidecar slot
Cargo.toml       — sysinfo, windows crate, tokio, serde, serde_yaml

cargo check: 0 errors, 2 expected dead_code warnings (Sprint 03 structs)

The PowerShell worker, pm2 ecosystem, Express HTTP server, and all
elevation-dependent infrastructure are now retired. AEGIS is a real
native Windows application.
